* Audit Fixes

** DONE [#A] Return error when library root cannot be canonicalized
:PROPERTIES:
:CATEGORY: security
:END:

=browse.rs:114= falls back to the raw =library_root= when =canonicalize=
fails.  The subsequent =starts_with= check compares a canonicalized target
against a possibly non-canonical root, which could allow a path traversal
bypass under certain filesystem conditions (NFS hiccup, permissions error).

Return =500 Internal Server Error= instead of silently degrading the
security check.

** DONE [#B] Log warnings on info.json parse failures
:PROPERTIES:
:CATEGORY: observability
:END:

=browse.rs:301-306= silently discards =info.json= read and parse errors
via =let Ok(...) = ... else { return default }=.  A truncated or malformed
sidecar (common after interrupted yt-dlp runs) produces a video entry with
no metadata and no log message for the operator to diagnose.

Add =warn!= log lines with the file path and the error on both the
=read_to_string= and =serde_json::from_str= failure branches.

** DONE [#C] Add integration tests for the path traversal check
:PROPERTIES:
:CATEGORY: testing
:END:

The existing =test_browse_path_traversal_rejected= covers the simple =..=
case.  Add tests for percent-encoded traversal (=%2e%2e=), nested traversal
(=sub/../../..=), and (on Unix) symlinks pointing outside the library root.

** IN-PROGRESS [#C] Make healthz verify library path accessibility
:PROPERTIES:
:CATEGORY: observability
:END:

=web_base.rs:59-63= always returns ={"status":"healthy"}= without checking
that =library_path= exists and is readable.  Call =fs::metadata= on the
library root and degrade to an unhealthy response when the path is
inaccessible.  This gives orchestrators and load balancers useful signal.
